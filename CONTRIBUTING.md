# Contributing to Blogify

Thank you for your interest in contributing! We welcome contributions to make this platform better.

## Code of Conduct
Please communicate with respect and professionalism.

## Getting Started

1. **Fork the repository** on GitHub.
2. **Clone your fork** locally.
3. **Install dependencies**: `npm install`
4. **Create a branch** for your feature: `git checkout -b feature/amazing-feature`

## Development Workflow

- **Linting**: Run `npm run lint` before committing.
- **Testing**: Ensure all tests pass with `npm test`. Add new tests for new features.
- **Commit Messages**: Use conventional commits (e.g., `feat: add comment notifications`, `fix: header alignment`).

## Pull Request Process

1. Push your changes to your fork.
2. Open a Pull Request (PR) to the `main` branch.
3. Describe your changes clearly in the PR description.
4. Wait for review and address any feedback.

## Style Guide

- We use **Prettier** for code formatting.
- **ESLint** is configured for linting.
- Follow the existing folder structure (modular architecture).
